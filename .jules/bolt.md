## 2024-04-24 - Optimizing List vs Set Lookups and String Iteration
**Learning:** Checking for exclusions in a list with `in` inside a loop results in O(N^2) complexity. Furthermore, doing string manipulation character by character in Python is quite slow compared to utilizing built-in C-optimized functions like `str.find()`. Converting `blacklist_data_exclusions` to a `set` drastically reduces blacklist processing time. Removing `list(parent)` in XML node matching avoids unnecessary list instantiations since `parent` is already an iterable of its children.
**Action:** When searching for matches in large text or strings, utilize built-in functions like `find` to fast-forward rather than doing `while` loops per character. Always convert exclusion lists to `set`s before doing `in` lookups. Avoid `list(iterable)` if only iterating is needed.
